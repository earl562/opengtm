# Security Policy

## Supported Versions
We support the latest `main` branch and the most recent release tag.

## Reporting a Vulnerability
Please **do not** open a public GitHub issue for security vulnerabilities.

Preferred reporting path:
1. Open a **GitHub Security Advisory** (private) in `earl562/opengtm`, or
2. Email: **security@opengtm.dev**

Include:
- Steps to reproduce
- Impact assessment
- Affected versions/commit

## Secrets & Credentials
Never include API keys, tokens, or customer data in issues, logs, or screenshots.
