## purpose
Review customer-facing messaging against the current positioning source so teams catch drift before it reaches prospects.

## when to use
Use when an operator wants feedback on an email, deck, one-pager, or messaging blurb.

## operator steps
1. Load the latest approved positioning document.
2. Compare the proposed message against core claims, differentiation, and language choices.
3. Flag statements that are off-positioning or unsupported.
4. Suggest targeted edits that bring the message back in line.

## gtm constraints and gotchas
- Annotate and suggest; do not rewrite the entire asset unless explicitly asked.
- Every flag should point back to a positioning section.
- Distinguish tone issues from factual positioning drift.
- Avoid softening critical gaps just to make the message pass.

## expected outputs
- A positioning review with cited flags, suggested edits, and an overall on-positioning judgment.
