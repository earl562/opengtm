## purpose
Assess whether the current product actually fits a prospect's stated needs using cited product documentation.

## when to use
Use during discovery follow-up, solution engineering support, or pre-demo qualification.

## operator steps
1. Extract the prospect's needs, constraints, and success criteria.
2. Search product docs for shipped capabilities that map to those needs.
3. Identify gaps, caveats, and acceptable workarounds.
4. Produce a fit score with citations and plain-language reasoning.

## gtm constraints and gotchas
- Never imply roadmap items are available now.
- Fit scoring should separate direct support from workaround-based support.
- Cite the doc source for each major capability claim.
- If the fit is weak, say so clearly so the deal team can reset expectations.

## expected outputs
- A cited feature-fit assessment with supported needs, gaps, workarounds, and overall fit score.
